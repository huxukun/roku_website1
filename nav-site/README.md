## 🚀 本地运行

```bash
# 进入项目目录
cd nav-site

# 安装依赖
npm install

# 启动开发服务器（Vite 默认 http://localhost:5173）
npm run dev

# 构建生产版本
npm run build
```

> **重要**：想要用手机测试定位/指南针功能，必须用 **HTTPS** 访问。
> 本地可使用 `https://localhost:5173` 或使用 Vercel 提供的预览链接。

---

## 🌐 部署到 Vercel

1. 把整个 `nav-site/` 目录推送到 GitHub
2. 在 Vercel 导入项目
3. 框架选择：**Vite**（或留空，它会自动识别）
4. 部署命令：`npm run build`
5. 输出目录：`dist`

Vercel 会自动分配一个 HTTPS 的预览链接，在手机浏览器打开即可测试。

## 🔑 API Key 配置（**必须做！**）

打开文件：[`src/config.js`](src/config.js)

```javascript
export const CONFIG = {

  // ---- 高德地图配置 ----
  AMAP_KEY: '你申请的高德 JS API Key',
  AMAP_SECURITY_CODE: '你申请的安全密钥',

  // ---- 天气 ----
  QWEATHER_KEY: '',   // 可选

  // ---- 模拟数据：没 Key 时也能看效果 ----
  USE_MOCK_DATA: false,

}
```

### 如何申请高德 Key？

1. 打开 https://console.amap.com/ 注册账号
2. 进入「应用管理」→「创建新应用」
3. 类型选择：**Web 端 (JS API)**
4. 复制 Key 和 securityJsCode 填入 `config.js`
5. 在「域名白名单」里加上你的 Vercel 部署域名

> ⚠️ 第一阶段的代码还**没有真正调用高德 API**（只是保留了位置），
> 所以即使你不填 Key，网站也能用（GPS + 指南针 + 语音都是浏览器原生能力）。

## 🎨 设计风格说明

- **背景**：纯黑 (`#000000`) — 在 AR 眼镜中显示为透明
- **主色**：青色荧光 (`#00ffff`) — 与绿色道路/红色信号灯形成对比
- **辅色**：绿色 (`#00ff88`) 用于安全提示/指南针北
- **文字**：大字号 + 大间距（骑行时一瞥即读）
- **动效**：慢速脉冲呼吸（不分散注意力）
- **3D 场景**：发光线框 + 粒子点（营造虚拟 HUD 感）

## ⚠️ 常见问题

**Q: 浏览器不请求定位权限？**
A: 请确认通过 HTTPS 访问，或 localhost。

**Q: 指南针不准？**
A: 手机需做"8字晃动"校准。室内金属环境会干扰。

**Q: 没有语音？**
A: 某些浏览器需要用户首次点击后才允许语音。我们的"开始导航"按钮已触发首次交互，之后会正常播报。

**Q: AR 眼镜投屏后看不到？**
A: 请确认投屏软件选择「全屏幕」，亮度调到最高。黑色背景应与真实世界融合。
